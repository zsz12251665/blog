Solution

This problem is very simple. You just need to find the small prime, and you'll get the big prime. So the solution will talk about how to check whether the integer is a prime.

We need to try all the integers from 2 to ⌊√(n)⌋. Why √(n)? That is because if there is a division d which is bigger than √(n), then n/d must be an integer, and it must be smaller than √(n). If both n/d and d are bigger than √(n), then d(n/d), which equals n, must be bigger than √(n)2, which equals n too. That is impossible. So you can check whether the integer n is a prime by trying all the integers from 2 to ⌊√(n)⌋.

P. S. In fact, we don't need to check whether the number is a prime. Because the problem have promised that n will always be the product of two primes. So we just need to find the small number, and output their quotient.

function prime(n:longint):boolean;
var
    i:longint;
begin
    for i:=2 to trunc(√(n)) do//Attention that the loop is from 2 because 2 is the smallest prime
        if n mod i=0 then//If there is a division, exit false
            exit(false);
    exit(true);
end;
begin
    read(n);
    for i:=2 to n do
        if prime(i)and(n mod i=0) then//You can replace this line with "if n mod i=0 then"
            begin
                write(n div i);
                halt;
            end;
end.
