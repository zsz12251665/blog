Solution

I spent a large amount of time thinking how to solve the problem with greedy algorithm or dp algorithm until I knew that the correct solution is halving searching algorithm since the answer is monotonical. Greedy algorithm requires an O(#n#(^2^)) time complexity and dp algorithm is completely wrong.

The halving searching algorithm is acceptable because it provides an O(#n#lg(#l#)) time complexity. First of all, we set the range as [0,#l#]. Then we try if making the minimum distance between two rocks as the middle value of the range is acceptable. Then we can reduce the searching range as half of the last range. Continue to take these steps until the range is empty. So we will only try lg(#l#) times. For each try, we can finish it by scanning the rocks once, which is O(#n#). So we solve the problem.
