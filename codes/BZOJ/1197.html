Solution

The meeting with the problem was during an in-team competition, the problems of which are from HNOI. It was the only one which I had the idea on what the problem is. But I had no idea on the solution. After the competition, one of my friends advised me to write out every pair of o{(}v{n}o{,}v{m}o{)}. And I did it. Then I had a sheet like this:

o{(}v{n}o{,}v{m}o{)} v{m}o{=}c{0} v{m}o{=}c{1} v{m}o{=}c{2} v{m}o{=}c{3} v{m}o{=}c{4} v{m}o{=}c{5} v{m}o{=}c{6} v{m}o{=}c{7} v{m}o{=}c{8} v{m}o{=}c{9} v{m}o{=}c{10}
v{n}o{=}c{1} c{1} c{2} c{4} c{6} c{8} c{10} c{12} c{14} c{16} c{18} c{20}
v{n}o{=}c{2} c{1} c{2} c{4} c{8} c{14} c{22} c{32} c{44} c{58} c{74} c{92}
v{n}o{=}c{3} c{1} c{2} c{4} c{8} c{16} c{30} c{52} c{84} c{128} c{186} c{260}

According to the limited data, we can find some interesting rules. First of all, it is obvious that the number of kinds of flowers always equals 1 when v{m}o{=}c{0}. Besides, if v{n}o{=}c{1} and v{m}o{≥}c{1}, the number of kinds of flowers is 2 times as large as v{m}. What's more, when v{n}o{≥}c{2} and v{m}o{≥}c{1}, the number of kinds of flowers equals the sum of the one with o{(}v{n}o{-}c{1}o{,}v{m}o{-}c{1}o{)} and the one with o{(}v{n}o{,}v{m}o{-}c{1}o{)}. And that's the solution! I don't know how to prove it, but the when I submitted the code with this solution, it did return an AC.
