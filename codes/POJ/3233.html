Solution

This problem is about the matrix multiplication. For two #n#*#n# matrix #A# and #B#, their product #P# can be calculated according to #P#(_#i#,#j#_)=∑(_#c#∈[1,#n#]_)(A(_#i#,#c#_)B(_#c#,#j#_)). For example, the product of two matrix [@#a#(_11_)@#a#(_21_)@ @#a#(_12_)@#a#(_22_)@] and [@#b#(_11_)@#b#(_21_)@ @#b#(_12_)@#b#(_22_)@] is [@#a#(_11_)#b#(_11_)+#a#(_12_)#b#(_21_)@#a#(_21_)#b#(_11_)+#a#(_22_)#b#(_21_)@ @#a#(_11_)#b#(_12_)+#a#(_12_)#b#(_22_)@#a#(_21_)#b#(_12_)+#a#(_22_)#b#(_22_)@]. So we can calculate the product of two matrix in O(#n#(^3^)). But how should we calculate their sum? As we all know, the matrix multiplication has the associative property, so we'll have#S#=∑(_#i#∈[1,#k#]_)(A(^#i#^))=/A(^#i#+1^)-#I#/#A#-#I#/-#I#, which #I# is the identity matrix. However, it is difficult to calculate the matrix division. So we need to think another way.

The best method is, if we know ∑(_#i#∈[1,2(^#t#^)]_)(A(^#i#^)), then ∑(_#i#∈[1,2(^#t#+1^)]_)(A(^#i#^)) would be ∑(_#i#∈[1,2(^#t#^)]_)(A(^#i#^))+#A#(^2(^#t#^)^)∑(_#i#∈[1,2(^#t#^)]_)(A(^#i#^)). So we may be able to convert #k# into binary, and then add the sum according to the binary. That makes the solution.
